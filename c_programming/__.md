## macro with temp capture variable

The __ variable captures the result so it can be used twice

Why __ as the name:
Double underscore is convention for "internal/temporary variable" - reduces chance of naming conflicts with variables in the surrounding code.

```c
CHECK_LOGE(i2c_master_start(handle), "Failed to start I2C");
//Expands to
esp_err_t __;
if ((__ = i2c_master_start(handle)) != ESP_OK) {
    PORT_EXIT_CRITICAL();
    ESP_LOGE(TAG, "Failed to start I2C");
    return __;  // Returns the actual error code from i2c_master_start
}
```

Without capturing
```c
if (some_function() != ESP_OK) {
    return some_function();  // BAD: calls function TWICE
}
```

With capturing
```c
esp_err_t __ = some_function();
if (__ != ESP_OK) {
    return __;  // GOOD: returns the same error code
}
```
